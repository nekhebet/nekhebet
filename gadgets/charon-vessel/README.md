# Charon Vessel — File Rotation Daemon

![C++17](https://img.shields.io/badge/C++-17-blue)
![Platform](https://img.shields.io/badge/platform-Linux%20%7C%20FreeBSD-blue)
![Single Binary](https://img.shields.io/badge/single-binary-lightgrey.svg)

Charon Vessel — демон ротации файлов для Linux и FreeBSD
с явным управлением жизненным циклом файлов.

Проект реализован в виде одного самодостаточного бинарного файла
и не использует внешние runtime-зависимости.


## Возможности

- Ротация файлов с атомарными операциями файловой системы
- Механизмы защиты от race-condition атак (TOCTOU, symlink, hardlink)
- Проверка и строгая валидация путей и имён файлов
- Поддержка cross-device перемещений файлов
- Параллельная обработка операций копирования
- Сбор внутренних метрик состояния и операций


## Принципы реализации

- Явное управление состояниями файлов
- Отказ при обнаружении небезопасных условий
- Минимальный набор доверенных компонентов
- Отсутствие фоновой или скрытой логики
- Использование системных примитивов ОС


## Ограничения и допущения

- Для запуска могут требоваться права root
- Конфигурация осуществляется через переменные окружения
- Рабочие директории должны существовать до запуска
- Имена файлов с вхождением `..` отклоняются намеренно
- FreeBSD: отсутствует поддержка `io_uring` и расширенных атрибутов


## Сборка

### Linux
```bash
g++ -std=c++17 -O3 -pthread -lcap -luring \
    charon_vessel.cpp -o charon_vessel
````

### FreeBSD

```bash
clang++ -std=c++17 -O3 -pthread -DNDEBUG \
    -Wall -Wextra -Wpedantic \
    charon_vessel.cpp -o charon_vessel
```


## Запуск

Минимальная конфигурация:

```bash
export FRESH_PATH="/var/charon/fresh"
export ARCHIVE_ROOT="/var/charon/archive"
export MAX_FRESH=100
```

Запуск демона:

```bash
./charon_vessel start
```

Перезагрузка конфигурации:

```bash
kill -HUP <pid>
```


## Конфигурация

Поведение демона настраивается через переменные окружения,
включая пути, лимиты, параметры параллелизма и проверки безопасности.

Полный список параметров описан в документации и структуре `Config`
в исходном коде.


## Запуск в виде сервиса (опционально)

Charon Vessel может быть запущен как системный сервис,
но это не является обязательным требованием.


## Мониторинг

Команда:

```bash
charon_vessel status
```

возвращает текущее состояние и внутренние счётчики операций.

Полный список метрик описан в документации.


## Безопасность

Charon Vessel предназначен для обеспечения корректного и
предсказуемого поведения при работе с файловой системой
в доверенной среде.

Модель угроз и ограничения описаны в `docs/SECURITY.md`.
