## Управление и интерфейс

### Архитектура отображения

Экран состоит из трёх логических слоёв:

1. **Центральный текст** — сообщения из выбранных источников Telegram в режиме реального времени
2. **Фоновые медиа** — фото, видео, lottie-анимации из выбранных источников Telegram
3. **Облако слов** — формируется из ранее опубликованных сообщений пользователя или канала 

Слои синхронизированы по времени и состояниям паузы.

### Кнопки управления

| Кнопка       | Пример отображения            | Действие                                                                 | Визуальные состояния                              |
|--------------|-------------------------------|--------------------------------------------------------------------------|----------------------------------------------------|
| **AUTO**     | AUTO (6) 🟢<br>AUTO (0) 🟡    | Вкл/выкл автосмену медиа каждые ≈5–8 сек                    | 🟢 — активно<br>🟡 — пауза<br>цифра — секунды до смены |
| **NEXT**     | ▶ NEXT                        | Ручной переход к следующему медиа                                 | подсвечивается при нажатии                         |
| **RESET**    | ↻ RESET (3)                   | Полный сброс: очищает очередь, переподключает соединение | красный индикатор 6 сек<br>цифра — сообщений в очереди |

### Жесты

- **Свайп влево** по области медиа → то же, что кнопка NEXT  
- **Клик по облаку слов** → **полная пауза** (медиа + текст + анимации). Повторный клик — возобновление.

### Типичный сценарий за первые 60 секунд

1. Запуск → показываются **выбранный текст** из файла (демо-режим, ~4 сек на фразу)  
2. Когда приходят сообщения:  
   - Выводится текст 
   - Выделяются ключевые слова
   - После текста выводится **облако слов**  
3. Каждые ≈5.5–8 сек — автоматический переход к следующему сообщению

## Визуальная сигнализация при получении важных событий

При поступлении сообщения или медиа с меткой приоритета включается **визуальная индикация**:

- **Двойная вспышка экрана** (случайный цвет + два быстрых fade-in/out)  
- **Временная смена радиального градиента фона** (на 3–5 секунд, разные оттенки в зависимости от метки)


## Облако слов (Word Cloud)

После вывода текста появляется **облако слов**.

### Алгоритм генерации облака слов

1. Запрос последних 100 сообщений от sender_id или chat_id
2. Объединение текста → toLowerCase()
3. Очистка:
   • удаление URL
   • удаление цифр
   • удаление пунктуации
4. Разбиение на слова (> 2 символов)
5. Удаление стоп-слов 
6. Подсчёт частоты
7. Сортировка по убыванию → топ-50
8. Формат: [{word: "ключевое", freq: 12}, ...]

| Ситуация                        | Источник слов                                              | Размер шрифта пропорционален |
|---------------------------------|------------------------------------------------------------|-------------------------------|
| Демо-режим (локальные фразы)    | Слова из всех фраз + запасной набор fallback-слов          | случайной частоте             |
| Нет данных                      | Случайный набор слов (WORD, CODE, DEPLOY и т.п.)           | —                             |


## Алгоритмы анализа текста (влияют на клиент)

| Алгоритм                  | Где выполняется | Что делает                                                                 | Результат передаётся клиенту в поле |
|---------------------------|-----------------|----------------------------------------------------------------------------|--------------------------------------|
| **Проверка special слов**   | ingest (Python) | Сравнение с глобальным словарем → список слов         | `tags.spec_words`                     |
| **NER** | ingest (natasha) | Извлечение персон, организаций, локаций                                   | передаётся или логируется |
| **Генерация word cloud**  | server (Node.js)| Частотный анализ + стоп-слова → топ-50 слов                               | `wordCloud`                          |
| **Подсветка слов**        | клиент (JS)     | Поиск точных вхождений слов из `tags.spec_words` в тексте                  | визуальная анимация   |

## Клиент ↔ Сервер:

### Основной поток событий

1. **Ingest-слой** (Python) получает сообщение из Telegram  
2. Скачивает медиа (если есть) → подписывает событие → сохраняет в базу  
3. База данных отправляет уведомление (LISTEN/NOTIFY)  
4. **Сервер** (Node.js) получает уведомление  
5. Проверяет подпись, генерирует облако слов, синхронизирует медиа-файлы  
6. Формирует payload и рассылает всем подключённым клиентам через WebSocket

### Что передаётся по WebSocket (два основных типа)

- **Сообщение**  
  - текст  
  - дата  
  - облако слов (массив {word, freq})  
  - метка приоритета (если есть)

- **Новое медиа**  
  - id файла  
  - тип файла  
  - время изменения  
  - метка приоритета (если есть)

### Медиа

- **При получении нового файла из Telegram:**  
  - Удаляется вся EXIF-метаинформация
  - Видео перекодируется в H.264 + AAC (если нужно)
  - Файл переименовывается в UUID
  - Записывается в таблицу БД
 
- **Через WebSocket приходит уведомление о новом файле**
  - Клиент добавляет его в очередь (с приоритетом, если указана метка)  
  - Периодически запрашивает список файлов по HTTP (если WS отвалился)  
  - Медиа загружаются по прямым ссылкам `/media/:id`

### Ротация медиа

- Есть активная директория для свежих файлов (ограничение по количеству)  
- При превышении лимита старые файлы перемещаются в архивную директорию  

### Защита от перегрузки

- Троттлинг сообщений (не чаще 500 мс на клиента)  
- Проверка размера буфера WebSocket (если большой — пропуск)  
- Регулярный heartbeat (каждые 30 сек) → отключение «мёртвых» клиентов

Nekhebet Omen Display предназначен для
длительной автономной работы
в режиме непрерывного отображения событий.
